# 프로젝트 I — 15일차 개발 일지

단계: Phase 1 — 프로토타입(싱글 핵심 루프)

## 목표

플레이어가 부활하지 못하고 최종 사망하거나 제한 시간 초과로 던전에 유기되었을 때, 인벤토리에 있던 모든 아이템과 보물을 현재 위치에 드랍하도록 구현한다.

## 한 일

### 1. 전체 소지품 드랍 기능 추가

- `InventorySystem.DropAll(Vector3)` 메서드 추가
- 일반 슬롯 아이템과 두손 운반 보물을 한 번에 인벤토리에서 제거
- 지정된 월드 위치 주변에 모든 소지품 배치
- 드랍된 소지품의 Renderer, Collider, Rigidbody 재활성화
- 아이템마다 임의 위치와 바깥쪽 힘을 적용하여 겹침 완화

### 2. 플레이어 최종 사망 연동

- `PlayerController.HandleDeath()`에서 전체 드랍 실행
- 부활의 돌이 없을 때만 모든 소지품 드랍
- 사망한 플레이어의 위치를 드랍 기준점으로 사용
- 사망 후 추가 피해에 의한 중복 드랍 방지

### 3. 부활의 돌 예외 처리

- 부활의 돌이 있으면 기존처럼 부활석 1개만 소모
- 자동 부활한 경우 다른 아이템과 보물 유지
- 부활 후 다시 사망하면 남아 있는 모든 소지품 드랍

### 4. 제한 시간 유기 연동

- `DungeonTimeSystem.OnDeadline()`에서 유기 판정 시 전체 드랍 실행
- 마차에서 멀리 떨어진 플레이어의 현재 위치에 소지품 드랍
- 드랍 처리가 끝난 뒤 `Time.timeScale`을 0으로 변경
- 마차 근처 자동 탈출 성공 시 기존처럼 소지품을 마차에 확보

### 5. 드랍 설정 추가

- `Drop Radius`로 아이템 배치 반경 설정
- `Drop Impulse`로 아이템이 튀어나가는 힘 설정
- 기본값을 각각 `0.8`, `2.5`로 지정

### 6. 코드 주석과 형식 정리

- `InventorySystem.cs`와 `DungeonTimeSystem.cs`의 의미 있는 코드 줄에 한글 주석 추가
- 한 줄 조건문과 프로퍼티를 Allman 스타일로 정리
- 클래스 설명을 현재 구현 상태에 맞게 수정

## 결과

- 최종 사망 시 인벤토리와 두손 소지품이 모두 월드에 드랍된다.
- 드랍 후 인벤토리 슬롯과 두손 운반 상태가 초기화된다.
- 부활의 돌이 작동한 경우 다른 소지품은 유지된다.
- 제한 시간 유기 시 플레이어 위치에 모든 소지품이 드랍된다.
- 마차 근처에서 제한 시간이 끝난 경우 드랍하지 않고 정상 탈출 처리된다.
- 사망 후 추가 공격을 받아도 소지품이 중복 생성되지 않는다.
