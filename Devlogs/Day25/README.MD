# 25일차 : 싱글 수직 슬라이스 통합 검증

---

## 목표

- 지금까지 구현한 시스템을 하나의 플레이 흐름으로 통합한다.
- 던전 생성부터 익스트랙션까지 전체 진행을 검증한다.
- 필수 오브젝트 누락과 자동 생성 실패를 확인할 수 있는 검증 도구를 구현한다.
- Windows 테스트 빌드에서 핵심 플레이가 작동하는지 확인한다.

---

## 한 일

- VerticalSliceValidator를 구현했다.
- 던전 생성기, 제한시간, 마차, 플레이어 및 스폰 매니저의 존재 여부를 검사하도록 했다.
- 생성된 방 수와 시작 방 존재 여부를 검사하도록 했다.
- 일반 몬스터, 보물, 함정의 자동 생성 결과를 확인하도록 했다.
- AudioListener 누락과 중복을 검사하도록 했다.
- 현재 몬스터, 보물, 미믹, 함정 및 기믹 몬스터 수를 표시하도록 했다.
- 마차 적재 개수와 총 가치, 남은 제한시간을 표시하도록 했다.
- 탈출 성공, 탈출 실패, 플레이어 사망 상태를 검증 패널에 표시하도록 했다.
- F8키로 현재 상태를 다시 검증할 수 있도록 했다.
- DungeonTimeSystem의 유기 실패 상태를 외부에서 확인할 수 있도록 확장했다.
- 고정 시드를 이용해 반복 가능한 던전 테스트를 진행했다.
- 확률형 미믹, 스토커, 고스트, 웃는 석상을 강제로 출현시켜 검증했다.
- 탐색, 전투, 보물 획득, 마차 적재 및 탈출 흐름을 검증했다.
- 함정, 상태 이상, 회복 아이템, 사망 및 부활 흐름을 검증했다.
- 마차 근처와 던전 내부에서 각각 제한시간 종료를 검증했다.
- 최종 확률과 제한시간을 기본값으로 복구했다.
- Windows Development Build를 생성해 전체 플레이 흐름을 확인했다.

---

## 결과

- 던전 생성부터 보물 확보와 탈출까지 하나의 플레이 흐름이 완성됐다.
- 필수 시스템이 누락되면 검증 패널과 Console에서 확인할 수 있다.
- 자동 생성 결과를 실행 중에 확인할 수 있다.
- 마차 적재 보물과 인벤토리 보물이 탈출 결과에 함께 반영된다.
- 시간 초과 시 마차 거리로 탈출 성공과 유기 실패가 구분된다.
- 사망, 부활, 상태 이상과 회복 아이템이 전체 흐름 안에서 작동한다.
- Windows 테스트 빌드에서 싱글 수직 슬라이스를 실행할 수 있다.

---

## 현재 제한 사항

- 결과 화면과 검증 패널은 임시 OnGUI 방식이다.
- 마을 정산과 빚 상환 시스템은 아직 연결하지 않았다.
- 몬스터 이동은 NavMesh를 사용하지 않아 벽과 꺾인 통로에서 막힐 수 있다.
- 정식 모델, 애니메이션, 사운드와 VFX가 부족하다.
- 자동 생성 오브젝트에 풀링을 적용하지 않았다.
- Windows 테스트 빌드는 개발 검증용이며 배포용 설정이 아니다.
- 멀티플레이어 기능과 네트워크 동기화는 구현하지 않았다.
