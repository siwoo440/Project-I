\# 34일차 개발 일지



\## 작업 주제



캠페인 성공 및 실패 결과 화면 구현



\## 작업 목표



\- 빚 전액 상환 시 캠페인 성공 결과 표시

\- 상환 기한 초과 시 캠페인 실패 결과 표시

\- 캠페인 최종 기록 표시

\- 결과 화면 중 플레이어와 마을 진행 정지

\- 결과 화면에서 메인 메뉴 이동

\- 결과 화면에서 새 캠페인 시작

\- 결과 화면에서 게임 종료



\## 구현 내용



\### CampaignEndingUI



Village Scene에서 CampaignManager의 캠페인 성공과 실패 상태를 감지하는 결과 화면을 구현했다.



결과 화면에는 다음 정보가 표시된다.



\- 성공 또는 실패 결과

\- 마지막 탐험 날짜

\- 완료한 탐험 횟수

\- 최종 보유 골드

\- 최종 남은 빚



임시 UI는 기존 프로젝트 규칙에 맞춰 OnGUI로 구현했다.



\### 결과 화면 진행 정지



결과 화면이 나타나면 다음 상태를 적용했다.



\- Time.timeScale을 0으로 변경

\- PlayerController 비활성화

\- PlayerInteractor 비활성화

\- 마우스 잠금 해제

\- 마우스 커서 표시

\- 기존 캠페인 HUD 숨김



Village Player가 늦게 생성되는 경우에도 결과 화면이 열린 동안 계속 검색하여 조작을 차단하도록 구현했다.



\### 메인 메뉴 이동



결과 화면에서 MainMenu Scene으로 돌아갈 수 있도록 구현했다.



메인 메뉴로 이동할 때 게임 시간과 마우스 상태를 복구하며 기존 캠페인 HUD가 메인 메뉴 위에 나타나지 않도록 처리했다.



완료된 캠페인 저장 기록은 유지되어 이어하기를 통해 결과를 다시 확인할 수 있다.



\### 새 캠페인 시작



결과 화면에서 새 캠페인을 선택하면 GameStartRequest에 NewGame 요청을 등록하고 Village Scene을 다시 불러오도록 구현했다.



새 캠페인에서는 다음 상태가 초기화된다.



\- 날짜

\- 보유 골드

\- 남은 빚

\- 완료 탐험 횟수

\- 캠페인 성공과 실패 상태

\- 상점 재고

\- 구매 대기 물자

\- 기존 저장 기록



\### 캠페인 HUD 제어



CampaignManager에 SetHudVisible 메서드를 추가했다.



Village 게임 시작 시에는 HUD를 표시하고 결과 화면과 MainMenu에서는 HUD를 숨기도록 연결했다.
