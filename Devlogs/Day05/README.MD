# 프로젝트 I — 5일차 개발 일지

> **단계**: Phase 1 — 프로토타입 (싱글 핵심 루프)
> **목표**: CSV → ItemData 자동 임포트 파이프라인 구축 (기획서 PART 13.2.2)

---

## 오늘의 목표
아이템 데이터를 **시트(CSV)로 관리 → 유니티로 일괄 임포트**하는 에디터 툴을 만들어, 밸런싱을 코드 수정 없이 반복할 수 있게 한다.

## 한 일

### 1. 데이터 위치 정리
- 데이터 원본은 `D:\프로젝트 I\데이터`, 유니티 프로젝트는 `F:\Project I` (드라이브 상이)
- → CSV를 **프로젝트 내부 `Assets/Data/아이템.csv`로 복사**해 정본으로 사용 (레포에 함께 버전관리)
- 앞으로 데이터 수정은 프로젝트 내 CSV를 편집 → 재임포트

### 2. `보너스칸` 컬럼 추가
- 가방류의 추가 인벤토리 칸(가방=4)을 데이터로 표현하기 위해 CSV에 컬럼 추가
- 정의서·기획서(PART 8.6) 동기화

### 3. ItemDataImporter (에디터 툴)
- 메뉴 **`ProjectI → Import → Items from CSV`**
- `Assets/Data/아이템.csv` 읽어 → `Assets/ScriptableObjects/Items/`에 **ItemData 에셋 21개 생성/갱신**
- **헤더 이름 기반 컬럼 매핑** (컬럼 순서 바뀌어도 동작)
- 기존 에셋이 있으면 갱신(중복 생성 방지), 없으면 생성
- 숫자 파싱은 InvariantCulture로 안전 처리

## 결과
- 아이템 21종 ItemData 에셋 자동 생성 확인
- CSV 값 수정 → 재임포트 시 에셋 갱신 확인 → **"시트만 고치면 반영"** 파이프라인 완성
