# 프로젝트 I — 11일차 개발 일지

> **단계**: Phase 1 — 프로토타입 (싱글 핵심 루프)
> **목표**: 몬스터 기본 AI (기획서 PART 7.3~7.5)

---

## 오늘의 목표
어두운 던전에 실제 위협 등장. 기본 FSM(배회→탐지→추격→공격→복귀) + 시각·청각 탐지 + 밝기 연동 구현.

## 한 일

### 1. MonsterData
- 몬스터 핵심 스탯 정의(ScriptableObject): 체력/이속/공격력/사거리/쿨다운/시야거리·각도/청각거리/어둠속공격성
- ※ 몬스터.csv는 컬럼이 복잡(다중 공격력 등)해 전용 CSV 임포터는 이후 별도 작성 — 지금은 수동 생성 또는 fallback 값 사용

### 2. MonsterAI (스텁 → 완성)
- **FSM**: Patrol(배회) → Chase(추격) → Attack(공격), 놓치면 배회 복귀
- **시각 탐지**: 시야각·거리 + 레이캐스트로 가림(벽) 판정
- **청각 탐지**: 플레이어 CharacterController 속도(=소음) 기반 — 가만히 있으면 안 들키고, 달리면 먼 거리에서도 감지
- **밝기 연동**: 플레이어의 `LightSystem.CurrentBrightness`가 낮으면(40 미만) 시야 탐지 범위 1.5배 확대(어두울수록 위험)
- **이동**: 단순 스티어링(직선 이동 + 회전 보간), CharacterController 기반 중력 적용
- **전투 스텁**: 사거리 내 쿨다운마다 플레이어 공격(`PlayerController.TakeDamage`), `TakeDamage()`로 피격 시 처치(Destroy) — 플레이어 공격 연동은 12일차 예정

## 결과
- 배회 중인 몬스터가 시야에 들어오면 추격, 사거리 도달 시 공격(체력 감소) 확인
- 조용히 걸으면 안 들키고 달리면 먼 거리에서 들키는 청각 탐지 확인
- 어두운 상태에서 탐지 범위가 넓어지는 밝기 연동 확인
- 시야 이탈 시 배회 복귀 확인
