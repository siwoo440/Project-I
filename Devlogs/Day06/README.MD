## 오늘의 목표
"빛을 관리해야 보인다"는 게임 정체성 구현. 씬을 칠흑으로 만들고, 광원(화톳불/랜턴)이 닿는 곳만 밝히며, **구역(방) 단위 밝기 수치**를 계산해 상단 UI에 5단계로 표시.

## 한 일

### 1. 씬 어둡게(URP)
- Environment: Skybox None, Ambient Color 검정, **Environment Reflections Intensity 0**
- Directional Light 비활성화, Main Camera **Background Solid Color 검정**
- → 광원 밖은 캄캄, 광원 반경만 시야 확보

### 2. LightSource (휴대 광원)
- 밝기 기여치(횃불 30 / 랜턴 50) + 실제 유니티 Light On/Off
- **초기값 꺼짐** → 손에 들고 **F로 켜고 끄기**

### 3. Brazier (화톳불 / 고정 광원)
- 방에 설치, 시간이 지날수록 밝기 **감소(decay)**, 조명 세기도 함께 감소
- **E로 재점화**(IInteractable) → 최대치 복구

### 4. LightRoom (구역 단위 밝기)
- 트리거 볼륨(Box). 최종 밝기 = 기본(0) + 방 안 화톳불 기여 합
- 플레이어가 들어오면 LightSystem에 현재 방 전달

### 5. LightSystem (통합, 스텁 → 완성)
- 최종 밝기 = **현재 방 고정 밝기 + 손에 든 광원 기여** (0~100 클램프)
- **밝기 5단계** 산출(매우 밝음~완전한 어둠), **상단 UI** 표시
- F 입력 처리(손 광원 토글)

## 결과
- 랜턴 없이는 거의 안 보이고, 랜턴을 들고 F로 켜면 앞이 밝아짐
- 방 이동 시 밝기 수치·단계 변화, 화톳불 감소/재점화 확인
