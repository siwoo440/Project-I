# 22일차 : 고스트·웃는 석상 및 위협 피드백

---

## 목표

- 플레이어 뒤쪽으로 순간이동해 기습하는 고스트를 구현한다.
- 플레이어 화면 노출 여부에 따라 행동하는 웃는 석상을 구현한다.
- 두 기믹 몬스터를 일반 몬스터와 분리된 규칙으로 출현시킨다.
- 위치 기반 3D 효과음 재생 구조를 구현한다.
- 몬스터의 경고, 출현, 공격, 피격을 소리와 파티클로 전달한다.
- 실제 효과음이 없어도 기능을 확인할 수 있는 임시 전자음을 구현한다.

---

## 한 일

### 고스트 구현

- 플레이어 뒤쪽의 안전한 바닥을 검색하는 Ghost를 구현했다.
- 플레이어 정후방이 막힌 경우 좌우 후방 위치를 추가로 검색하도록 했다.
- 순간이동 전에는 고스트의 외형과 충돌을 비활성화했다.
- 순간이동 후 일정 시간이 지나면 플레이어에게 기습 피해를 적용하도록 했다.
- 고스트와 플레이어 사이에 장애물이 있는지 검사하도록 했다.
- 플레이어가 기습 범위에서 벗어나면 기습 피해를 적용하지 않도록 했다.
- 기습 종료 후 기존 MonsterAI 추격으로 전환하도록 연결했다.
- 기존 MonsterAI의 체력, 방어력, 피격 및 처치 처리를 재사용했다.

### 웃는 석상 구현

- 플레이어 카메라를 기준으로 웃는 석상의 화면 노출 여부를 판정하도록 했다.
- 초기에는 카메라 중앙의 제한된 시선 각도를 이용해 판정했다.
- 테스트 과정에서 인식 범위가 지나치게 좁은 문제를 확인했다.
- 시선 각도 방식 대신 카메라 화면 전체 영역을 검사하도록 수정했다.
- 석상 자식 오브젝트의 모든 Renderer 영역을 하나의 Bounds로 결합했다.
- 결합된 석상 영역과 카메라 화면 영역이 겹치는지 검사하도록 했다.
- 석상의 중심, 위쪽, 아래쪽, 왼쪽, 오른쪽 지점을 이용해 노출 여부를 확인하도록 했다.
- 플레이어 자신의 Collider가 시야 검사에 포함되지 않도록 제외했다.
- 플레이어와 석상 사이의 벽과 장애물을 Raycast로 검사하도록 했다.
- 석상 일부만 화면 가장자리에 보여도 정지하도록 개선했다.
- 석상이 벽에 완전히 가려지면 화면 방향에 있어도 움직이도록 했다.
- 석상 전체가 화면 밖으로 사라지면 플레이어에게 접근하도록 했다.
- 웃는 석상이 화면에 보이는 동안 이동과 공격을 모두 정지하도록 했다.
- 공격 범위에서 플레이어가 시선을 돌리면 피해를 적용하도록 했다.
- 기존 무기 공격과 MonsterAI의 피격 및 처치 처리를 재사용했다.

### 기믹 몬스터 자동 출현

- 고스트와 웃는 석상을 관리하는 GimmickMonsterSpawnManager를 구현했다.
- 기존 일반 MonsterSpawnManager와 분리된 출현 구조를 적용했다.
- 고스트와 웃는 석상이 각각 독립된 확률로 출현하도록 했다.
- 시작 방을 자동 생성 후보에서 제외했다.
- 플레이어 주변을 자동 생성 위치에서 제외했다.
- DungeonGenerator의 GenerationCompleted 이벤트와 연결했다.
- DungeonSpawnUtility를 이용해 안전한 생성 위치를 검색하도록 했다.
- 던전 재생성 시 이전 기믹 몬스터를 제거하도록 했다.
- 자동 생성된 고스트와 웃는 석상 수를 OnGUI에 표시했다.

### 3D 효과음 기반

- 기존 AudioManager 스텁을 3D 효과음 재생 시스템으로 교체했다.
- 효과음 발생 위치와 거리에 따라 음량과 좌우 방향이 달라지도록 했다.
- Scene 전환 후에도 AudioManager가 하나만 유지되도록 구성했다.
- 실제 AudioClip이 없을 때 임시 전자음을 생성하도록 했다.
- 임시 AudioSource가 재생 완료 후 자동으로 제거되도록 했다.

### 공통 위협 피드백

- 공통 피드백 컴포넌트인 ThreatFeedback을 구현했다.
- 경고, 출현, 공격, 피격 피드백을 하나의 컴포넌트에서 관리하도록 했다.
- 경고, 충격, 고스트 출현용 임시 파티클 프리팹을 제작했다.
- MonsterAI의 공격과 피격 이벤트에 공통 피드백을 연결했다.
- 생성된 파티클이 재생 완료 후 자동으로 제거되도록 했다.

### 고스트 피드백

- 고스트 순간이동 전에 플레이어 뒤쪽에서 경고음을 재생하도록 했다.
- 순간이동 약 0.7초 전에 방향성 경고가 발생하도록 했다.
- 고스트 출현 시 회색 파티클과 출현음을 재생하도록 했다.
- 고스트 기습 공격과 피격에 공통 피드백을 연결했다.

### 웃는 석상 피드백

- 웃는 석상이 화면 밖으로 사라져 움직이기 시작할 때 경고음을 재생하도록 했다.
- 이동 시작 시 경고 파티클을 재생하도록 했다.
- 웃는 석상의 공격과 피격에 공통 피드백을 연결했다.
- 화면 노출 상태가 변경될 때 디버그 로그를 출력하도록 했다.

## 결과

- 고스트가 플레이어 뒤쪽의 안전한 위치로 순간이동한다.
- 플레이어 정후방이 막혀 있으면 다른 후방 위치를 검색한다.
- 고스트가 순간이동 전에 플레이어 뒤쪽에서 경고음을 재생한다.
- 고스트 출현 시 임시 소리와 파티클이 재생된다.
- 순간이동 직후 플레이어에게 기습 피해를 준다.
- 기습이 끝난 고스트는 기존 MonsterAI로 플레이어를 추격한다.
- 웃는 석상은 화면 중앙뿐 아니라 화면 가장자리에 보여도 정지한다.
- 웃는 석상의 몸 일부만 화면에 남아 있어도 움직이지 않는다.
- 웃는 석상 전체가 화면 밖으로 사라지면 접근을 시작한다.
- 웃는 석상이 벽에 완전히 가려지면 움직일 수 있다.
- 웃는 석상이 움직이기 시작할 때 경고 피드백이 재생된다.
- 웃는 석상을 다시 화면에 넣으면 현재 위치에서 정지한다.
- 두 기믹 몬스터 모두 기존 무기로 공격하고 처치할 수 있다.
- 일반 몬스터의 공격과 피격을 소리와 파티클로 확인할 수 있다.
- 실제 효과음 에셋이 없어도 임시 전자음으로 기능을 테스트할 수 있다.
- 모든 효과음이 3D 위치와 거리에 맞게 재생된다.
- 두 기믹 몬스터가 일반 몬스터와 독립된 확률로 출현한다.

## 현재 제한 사항

- 고스트와 웃는 석상은 NavMesh를 사용하지 않는다.
- 고스트의 기습 이후 추격은 벽과 꺾인 통로에서 막힐 수 있다.
- 웃는 석상은 플레이어 방향으로 직선 이동하므로 벽을 우회하지 못한다.
- 매우 얇은 틈으로 석상 일부만 보이는 상황에서는 화면 노출 판정이 달라질 수 있다.
- 임시 전자음과 기본 파티클은 최종 게임용 에셋이 아니다.
- 같은 효과가 짧은 시간에 반복되면 임시 오브젝트가 많이 생성될 수 있다.
- AudioSource와 파티클 오브젝트 풀링은 아직 적용하지 않았다.
- 함정과 환경 상호작용 피드백은 아직 공통 시스템에 연결하지 않았다.
- 정식 모델, 애니메이션, 카메라 흔들림은 구현하지 않았다.
- 멀티플레이어 목표 선택과 상태 동기화는 구현하지 않았다.
