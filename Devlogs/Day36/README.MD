\# 36일차 개발 일지



\## 작업 주제



게임 중 일시정지 메뉴와 탐험 포기 흐름 구현



\## 작업 목표



\- Esc 일시정지 메뉴 구현

\- 플레이어 이동과 전투 및 상호작용 정지

\- 기존 게임 설정 화면 연결

\- 던전 탐험 포기 기능 구현

\- 탐험 포기 후 기존 결과 및 정산 흐름 연결

\- Village 상태 저장 후 MainMenu 이동

\- 결과 화면과 일시정지 메뉴 충돌 방지



\## 구현 내용



\### PauseMenuUI



Village와 Dungeon에서 Esc 입력으로 열 수 있는 OnGUI 기반 일시정지 메뉴를 구현했다.



일시정지 중에는 다음 상태를 적용한다.



\- Time.timeScale 정지

\- PlayerController 비활성화

\- PlayerInteractor 비활성화

\- PlayerCombat 비활성화

\- 마우스 잠금 해제

\- 마우스 커서 표시



계속하기를 선택하면 일시정지 전 컴포넌트 활성 상태와 시간 및 마우스 상태를 복원한다.



\### 게임 설정 연결



35일차 GameSettingsUI를 PauseMenu 프리팹에 함께 배치했다.



일시정지 메뉴에서 설정 화면을 열 수 있으며 설정을 적용하거나 취소하면 다시 일시정지 메뉴로 돌아온다.



\### 탐험 포기



RunEndReason에 ManualAbandoned를 추가했다.



Dungeon에서 탐험 포기를 확정하면 RunResultManager가 기존 CompleteRun 흐름을 사용하여 실패 결과를 생성한다.



플레이어가 들고 있는 물품은 확보되지 않으며 마차에 적재한 물품만 기존 결과 계산에 포함된다.



결과 화면 표시 후 기존 설정에 따라 10초 뒤 Village로 이동한다.



\### 메인 메뉴 이동



Village 일시정지 메뉴에서 현재 상태를 저장하고 MainMenu로 이동할 수 있도록 구현했다.



빚 납부 선택이 진행 중이거나 저장에 실패한 경우에는 데이터 손실 방지를 위해 MainMenu 이동을 차단한다.



\### UI 충돌 방지



다음 화면이 열려 있을 때는 일시정지 메뉴가 나타나지 않도록 처리했다.



\- 던전 결과 화면

\- 캠페인 성공 화면

\- 캠페인 실패 화면

\- 빚 납부 선택 화면
