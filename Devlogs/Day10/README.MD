## 오늘의 목표
게임시간 6:00→24:00 카운트다운(현실 약 18분) 구현. 시간 초과 시 탈출 봉쇄(유기)로 처리해 익스트랙션의 시간 압박을 완성.

## 한 일

### 1. DungeonTimeSystem
- **게임시간 6:00 → 24:00** 을 `Real Duration Seconds`(기본 1080초=18분)에 걸쳐 진행
- `Progress`(0~1), `GameHour`, `RemainingSeconds`, `IsLocked`(24:00 도달) 제공
- 우측 상단에 **던전 시간(시:분) + 남은 시간(분:초)** 표시
- 24:00 도달(`OnDeadline`) 시 기본은 **탈출 봉쇄(유기, 실패)** 처리 → 화면 중앙에 실패 안내, `Time.timeScale = 0`
- 씬 시작 시 `Time.timeScale = 1f`로 초기화(이전 플레이의 종료/실패 정지 상태가 남지 않도록)

### 2. 시간대별 위협 증가 (기존 시스템 재확인)
- 화톳불(`Brazier`)이 시간이 지날수록 밝기가 줄어드는 것(6일차)이 곧 "시간 갈수록 위협 증가"의 구현체임을 확인
- 몬스터 강화·스폰 증가 연동은 몬스터(11~14일차)·랜덤 스폰(13일차) 이후 확장 예정

## 추가 작업 — 시간 초과 시 마차 근처면 탈출 성공
- **요청**: 시간이 다 됐어도 플레이어가 마차에 있으면 실패가 아니라 성공으로 처리
- **수정**: `OnDeadline()`에서 마차(`Wagon`)와 플레이어 인벤토리(`InventorySystem`) 위치를 확인해, **마차와의 거리가 `Wagon Proximity`(기본 5m) 이내면 `Wagon.Leave(playerInv)` 호출**로 탈출 성공 처리 후 종료
- 그 외의 경우에만 기존대로 유기(실패) 처리

## 결과
- 시간 흐름·표시 정상, 시간 내 떠나기(9일차) 정상
- **경계 케이스 확인**: 마차 옆에서 시간 초과 → 탈출 성공(마차+인벤 보물 확보) / 던전 안쪽에서 시간 초과 → 탈출 실패(유기) 확인
