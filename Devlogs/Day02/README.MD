## 오늘의 목표
걸어다니고 둘러볼 수 있는 **1인칭 캐릭터** + **체력·스태미너·낙하 피해** 기반 마련.

## 한 일

### 1. 테스트 맵 구성 (Dungeon 씬)
- 바닥(Plane) + 장애물/계단(Cube) + **낙하 테스트용 높은 발판** 배치

### 2. 플레이어 오브젝트
- Capsule → `Player` (Capsule Collider 제거, **Character Controller** 추가: Height 1.8 / Radius 0.4)
- 1인칭이라 **Mesh Renderer 비활성화**
- **Main Camera를 자식**으로, 눈높이 (0, 0.7, 0)에 배치

### 3. PlayerController.cs 작성 (`Assets/Scripts/Player`)
새 Input System 저수준 폴링(Keyboard/Mouse.current) 기반. 주요 기능:

| 기능 | 내용 |
| :-- | :-- |
| 이동 | WASD (걷기 4 m/s) |
| 시점 | 마우스룩 (몸통=요, 카메라=피치, ±85° 클램프) |
| 달리기 | Shift (전진 중, 7 m/s) — 스태미너 소모 |
| 점프 | Space — 스태미너 10 소모 |
| 앉기 | Ctrl — 높이/눈높이 낮춤, 속도 2 m/s |
| 체력 | 100 (기획서 기준값) + `TakeDamage()` |
| 스태미너 | 100, 달리기 소모 / 미사용 시 딜레이 후 자동 회복 |
| 낙하 피해 | 낙하속도 임계(10 m/s) 초과분에 비례 피해 |
| 편의 | Esc 커서 해제 / 클릭 재잠금, 좌상단 디버그 표시 |

- 수치는 전부 SerializeField로 노출 → Inspector에서 밸런싱 가능
- 모든 값 **잠정**(기획서 PART 4 잠정치 기준)

## 결과
- WASD 이동·마우스 시점·달리기(스태미너 감소)·점프·앉기·낙하 피해 정상 동작 확인
- 이후 모든 플레이어 상호작용의 토대 완성
