\# 33일차 개발 일지



\## 작업 주제



메인 메뉴와 새 게임 및 이어하기 흐름 구현



\## 작업 목표



\- 게임 실행 시 메인 메뉴 표시

\- 새 게임 시작 기능 구현

\- 기존 저장 기록 이어하기 구현

\- 새 게임 시작 전 기존 저장 기록 삭제 확인

\- Village 진입 후 캠페인 초기화 또는 저장 데이터 복원

\- 게임 종료 기능 구현



\## 구현 내용



\### MainMenu Scene



게임 실행 시 가장 먼저 열리는 MainMenu Scene을 추가했다.



임시 UI는 기존 프로젝트 규칙에 맞춰 OnGUI로 구현했으며 다음 버튼을 제공한다.



\- 새 게임

\- 이어하기

\- 게임 종료



저장 파일이 없으면 이어하기 버튼이 비활성화된다.



\### GameStartRequest



MainMenu에서 선택한 새 게임 또는 이어하기 요청을 Village Scene으로 전달하는 정적 시작 요청 시스템을 구현했다.



Village Scene에서 요청을 한 번 가져오면 요청을 초기화하여 Scene을 다시 방문했을 때 중복 실행되지 않도록 처리했다.



\### VillageGameStartLoader



Village Scene의 필수 관리자가 생성될 때까지 대기한 후 시작 요청을 처리하도록 구현했다.



새 게임을 선택하면 다음 상태를 초기화한다.



\- 날짜

\- 보유 골드

\- 남은 빚

\- 완료 탐험 횟수

\- 캠페인 성공과 실패 상태

\- 상점 재고

\- 다음 탐험 구매품



이어하기를 선택하면 32일차에 구현한 JSON 저장 기록을 불러온다.



\### 저장 기록 보호



기존 저장 파일이 있는 상태에서 새 게임을 누르면 즉시 삭제하지 않고 확인 화면을 표시하도록 구현했다.



플레이어가 삭제 후 새 게임을 확정했을 때만 기존 저장 기록을 삭제한다.



\### Build Profile



Scene 실행 순서를 다음과 같이 설정했다.



1\. MainMenu

2\. Village

3\. Dungeon
