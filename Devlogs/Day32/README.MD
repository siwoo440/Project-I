\# 32일차 개발 일지



\## 작업 주제



캠페인 저장 및 불러오기 시스템



\## 작업 목표



\- 게임 종료 후에도 캠페인 진행 상태 유지

\- 마을을 안전한 저장 지점으로 사용

\- 날짜와 골드 및 남은 빚 저장

\- 캠페인 성공과 실패 상태 저장

\- 상점 재고와 구매 대기 물자 저장

\- 빚 납부와 상점 구매 후 자동 저장

\- 마을 저장 장부를 통한 수동 저장과 불러오기



\## 구현 내용



\### CampaignSaveData



JsonUtility로 저장할 캠페인 데이터를 정의했다.



저장 항목은 다음과 같다.



\- 현재 날짜

\- 빚 상환 마감 날짜

\- 보유 골드

\- 남은 빚

\- 완료 탐험 횟수

\- 캠페인 성공 여부

\- 캠페인 실패 여부

\- 다음 탐험 구매품

\- 상점의 남은 재고

\- 저장 시간

\- 저장 파일 버전



\### CampaignSaveManager



Application.persistentDataPath 아래에 JSON 저장 파일을 생성하도록 구현했다.



마을에서만 저장과 불러오기를 허용하며 빚 납부 선택이 진행 중일 때는 불완전한 상태가 저장되지 않도록 차단했다.



중복 관리자 생성을 막기 위해 싱글톤과 DontDestroyOnLoad를 적용했다.



\### VillageSaveLedger



마을의 빚 납부 건물에 원정 장부를 배치했다.



플레이어가 장부와 상호작용하면 OnGUI 기반 저장 화면이 열리며 다음 기능을 사용할 수 있다.



\- 현재 진행 저장

\- 저장 기록 불러오기

\- 저장 기록 삭제

\- 저장 기록 요약 확인



저장 파일 삭제는 실수를 방지하기 위해 두 번 확인하도록 구현했다.



\### 자동 저장



다음 시점에 자동 저장하도록 연결했다.



\- 플레이어가 빚 납부를 확정한 직후

\- 마을 상점에서 아이템 구매가 완료된 직후



\### 상점 상태 저장



상점 아이템의 ScriptableObject 이름을 임시 저장 식별자로 사용했다.



상점의 남은 재고와 다음 던전에 전달할 구매품 목록을 함께 저장하고 복원하도록 구현했다.



\## 저장 규칙



\- 마을에서만 저장할 수 있다.

\- 빚 납부 선택 중에는 저장할 수 없다.

\- 던전 내부 진행 상황은 저장하지 않는다.

\- 던전 중 종료하면 마지막 마을 저장 지점으로 돌아간다.

\- 던전 출발 전에 구매한 물자는 저장된다.



\## 테스트 결과



\- 마을 저장 장부 상호작용 정상

\- JSON 저장 파일 생성 정상

\- 게임 재실행 후 캠페인 상태 복원 정상

\- 날짜와 골드 및 남은 빚 복원 정상

\- 상점 재고 복원 정상

\- 구매 대기 물자 복원 정상

\- 빚 납부 후 자동 저장 정상

\- 상점 구매 후 자동 저장 정상

\- 저장 파일 삭제 재확인 정상

\- Scene 왕복 후 저장 관리자 중복 방지 정상
