# 프로젝트 I — 8일차 개발 일지

> **단계**: Phase 1 — 프로토타입 (싱글 핵심 루프)
> **목표**: 보물 시스템 + 하단 인벤토리 핫바 UI (기획서 PART 8.3, 11.4.1)

---

## 오늘의 목표
보물 데이터 연동, **가치 생성 시 고정**(기본랜덤×밝기배수×리스크배수), **두손 운반** 보물, 그리고 인벤토리를 눈으로 볼 수 있는 하단 핫바 UI.

## 한 일

### 1. ICarryable (공용 인터페이스)
- 아이템·보물이 인벤토리에서 **동일하게 관리**되도록 공통 규격 도입 (DisplayName/Weight/Slots/BonusSlots/**TwoHanded** + 보관·표시·배출)
- `PickupItem`이 이를 구현하도록 리팩터링

### 2. TreasureData + Treasure
- `TreasureData`(ScriptableObject): 이름/최소가/최대가/무게/칸/**두손운반**/소음 (보물.csv 컬럼 대응)
- `Treasure`: **가치는 생성 시점에 고정** — `기본랜덤(min~max) × 밝기배수(생성 위치) × 리스크배수`
  - 이후 옮겨도 값 불변(악용 방지, 기획서 8.3.1)
  - 밝기배수는 보물이 놓인 `LightRoom`을 찾아 자동 반영(정밀한 스폰시점 연동은 13일차 예정)

### 3. TreasureDataImporter
- 5일차와 동일한 방식의 CSV→SO 임포터. 메뉴 `ProjectI → Import → Treasures from CSV`
- `보물.csv` → `Assets/ScriptableObjects/Treasures/` 112종 생성

### 4. InventorySystem 확장 — 두손 운반
- 두손 보물은 **슬롯 대신 양손 점유**로 관리(`twoHand` 필드), 휠 전환 잠금
- **두손 물건을 들고 있으면 Q로 내려놓기 전까지 아무것도 못 줍게 차단**

### 5. 하단 중앙 핫바 UI
- 인벤토리 칸을 하단 중앙에 시각화(임시 OnGUI), 선택된 칸 강조, 다중 슬롯 아이템은 칸 폭만큼 표시

### 6. 추가 보정 — 광원 소지 방식 개선
- **문제**: 인벤토리에 넣으면 오브젝트를 비활성화해, 손에서 내려놓은 랜턴이 꺼진 것처럼 보임
- **수정**: 보관 시 **오브젝트는 활성 유지 + 모델(Renderer)만 숨김** → 켜진 랜턴은 소지 중이면 계속 빛남
- `LightSystem`: 밝기 기여를 **소지한 모든 켜진 광원의 합**으로 계산(손+인벤토리), **F 토글은 '손에 든' 광원만** 작동

## 결과
- 보물 가치 표시(`[E] 보물: 이름 (N골드)`), 옮겨도 값 고정 확인
- 두손 보물 운반 중 다른 것 못 줍는 제약 확인
- 하단 핫바로 인벤토리 상태를 시각적으로 확인 가능
- 랜턴 On 상태로 손에서 내려도 빛 유지, F는 손에 든 것만 반응 확인
