# 23일차 : 플레이어 피격 및 함정 피드백

---

## 목표

- 플레이어 피해를 화면과 카메라 움직임으로 전달한다.
- 일반 피해와 즉사 피해의 피드백 강도를 구분한다.
- 가시·도끼·즉사 함정에 경고와 발동 피드백을 추가한다.
- 짧은 시간 동안 같은 피드백이 중복 재생되는 현상을 제한한다.

---

## 한 일

- PlayerController에 일반 피해량과 즉사 여부를 전달하는 Damaged 이벤트를 추가했다.
- PlayerDamageFeedback을 구현했다.
- 플레이어 피해량에 따라 붉은 화면 점멸 강도가 달라지도록 했다.
- 일반 피해와 즉사 피해의 카메라 흔들림 강도를 구분했다.
- 카메라 흔들림 전용 CameraShakePivot을 추가했다.
- 피격 효과음이 없을 때 임시 전자음을 재생하도록 했다.
- ThreatFeedback에 같은 피드백의 최소 재생 간격을 추가했다.
- 지정된 위치에서 공격 피드백을 재생하는 PlayAttackAt 기능을 추가했다.
- 기존 Trap에 ThreatFeedback을 연결했다.
- 플레이어가 함정 Trigger에 진입하면 경고 피드백을 재생하도록 했다.
- 실제 피해가 적용되는 순간 함정 발동 피드백을 재생하도록 했다.
- 가시, 도끼, 즉사 함정용 임시 파티클을 제작했다.
- 함정 종류에 따라 피드백 위치와 임시 전자음 주파수를 다르게 설정했다.

---

## 결과

- 플레이어가 피해를 받으면 화면이 붉게 점멸한다.
- 피해량이 높을수록 화면 점멸과 카메라 흔들림이 강해진다.
- 즉사 피해는 일반 피해보다 강한 피드백을 사용한다.
- 방패로 감소한 실제 피해량이 피드백 강도에 반영된다.
- 가시·도끼·즉사 함정이 발동 전 경고를 제공한다.
- 함정 발동 순간에 종류별 소리와 파티클이 재생된다.
- 부활의 돌을 사용한 즉사 피해에서도 피드백이 정상 작동한다.
- 짧은 시간에 동일한 피드백이 중복 재생되지 않는다.

---

## 현재 제한 사항

- 붉은 화면 점멸은 임시 OnGUI 방식이다.
- 카메라 흔들림은 위치 이동만 사용하며 회전 흔들림은 적용하지 않았다.
- 임시 전자음과 파티클은 최종 게임용 에셋이 아니다.
- AudioSource와 파티클 오브젝트 풀링은 아직 적용하지 않았다.
- 실제 함정 모델의 애니메이션과 피드백 재생 시점은 연결하지 않았다.
- 멀티플레이어 피격 이벤트 동기화는 구현하지 않았다.
