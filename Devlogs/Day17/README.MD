\# 17일차 : 밝기 연동 보물 자동 스폰 및 스폰 매니저 분리



\## 목표



\- 던전 생성 완료 후 각 방에 보물을 자동으로 배치한다.

\- 어두운 방일수록 더 많은 보물이 생성되도록 한다.

\- 방 밝기와 리스크 배율에 따라 보물 가치를 결정한다.

\- 몬스터와 보물 생성을 서로 독립된 매니저로 분리한다.

\- 몬스터와 보물이 안전한 위치에 생성되도록 공통 위치 검색 기능을 구성한다.



\## 한 일



\### 1. 밝기 연동 보물 자동 스폰 구현



\- 던전 생성 완료 이벤트를 받아 보물을 자동 생성하도록 구현했다.

\- `DungeonGenerator.PlacedRooms`를 이용해 생성된 모든 방을 순회했다.

\- 시작 방에는 보물이 생성되지 않도록 설정했다.

\- 방마다 생성할 기본 보물 수를 무작위로 결정하도록 구성했다.

\- 방이 어두울수록 보물 생성량이 증가하도록 밝기 배율을 적용했다.

\- 보물 프리팹 목록에서 생성할 보물을 무작위로 선택하도록 구현했다.

\- 생성된 보물이 `Treasure.GenerateValue()`를 통해 고정된 가치를 가지도록 처리했다.



\### 2. 보물 가치에 밝기 배율 적용



방의 기본 밝기에 따라 다음 배율을 적용했다.



| 방 밝기 | 배율 |

|---:|---:|

| 81 이상 | 0.5 |

| 61\~80 | 0.8 |

| 41\~60 | 1.0 |

| 21\~40 | 1.5 |

| 0\~20 | 2.0 |



\- 밝은 방에서는 보물 생성량과 가치가 감소한다.

\- 어두운 방에서는 보물 생성량과 가치가 증가한다.

\- 이후 리스크 시스템을 연결할 수 있도록 별도의 리스크 배율을 추가했다.



\### 3. 안전한 보물 생성 위치 검색



\- `LightRoom`의 Collider 영역을 기준으로 방 내부 위치를 검색했다.

\- 벽에서 일정 거리 이상 떨어진 위치만 사용하도록 했다.

\- 바닥을 향해 Raycast를 실행해 실제 바닥 높이를 찾도록 했다.

\- 플레이어 주변 안전거리 안에는 보물이 생성되지 않도록 했다.

\- 기존 몬스터, 보물, 벽과 겹치는 위치를 제외하도록 충돌 검사를 적용했다.

\- 제한된 횟수 안에 안전한 위치를 찾지 못하면 해당 생성을 건너뛰도록 처리했다.



\### 4. LightRoom Trigger 충돌 문제 수정



몬스터와 보물이 모두 생성되지 않는 문제를 확인했다.



원인은 `Physics.CheckSphere()`가 방 전체를 감싸는 `LightRoom`의 Trigger Collider를 장애물로 감지하는 것이었다.



다음 물리 검사에서 Trigger Collider를 제외하도록 수정했다.



\- 바닥 검색용 `Physics.Raycast()`

\- 생성 위치 충돌 검사용 `Physics.CheckSphere()`



`QueryTriggerInteraction.Ignore`를 사용해 `LightRoom`은 밝기 판정 영역으로만 사용하고 실제 스폰 장애물로 판단하지 않도록 변경했다.



\### 5. SpawnManager 역할 분리



기존 `SpawnManager`가 몬스터와 보물 생성을 모두 담당하던 구조를 분리했다.



새로운 구조는 다음과 같다.



```text

DungeonGenerator

└─ GenerationCompleted

&#x20;  ├─ MonsterSpawnManager

&#x20;  └─ TreasureSpawnManager



추가한 스크립트:



Assets/Scripts/Managers/MonsterSpawnManager.cs

Assets/Scripts/Managers/TreasureSpawnManager.cs

Assets/Scripts/Managers/DungeonSpawnUtility.cs



제거한 기존 스크립트:



Assets/Scripts/Managers/SpawnManager.cs

6\. MonsterSpawnManager 구현

던전 생성 완료 이벤트를 구독한다.

생성된 방을 순회하며 몬스터를 자동 배치한다.

시작 방의 몬스터 생성을 제외한다.

방 밝기에 따라 몬스터 생성 수를 조절한다.

생성한 몬스터를 별도의 목록으로 관리한다.

던전 재생성 시 기존 자동 생성 몬스터를 제거한다.

현재 자동 생성 몬스터 수를 임시 OnGUI로 표시한다.

7\. TreasureSpawnManager 구현

던전 생성 완료 이벤트를 구독한다.

생성된 방을 순회하며 보물을 자동 배치한다.

시작 방의 보물 생성을 제외한다.

방 밝기에 따라 보물 생성량과 가치를 조절한다.

생성한 보물을 별도의 목록으로 관리한다.

던전 재생성 시 기존 자동 생성 보물을 제거한다.

현재 자동 생성 보물 수를 임시 OnGUI로 표시한다.

8\. DungeonSpawnUtility 구현



몬스터와 보물 매니저에서 공통으로 사용하던 기능을 별도 유틸리티로 분리했다.



공통 기능:



방 밝기에 따른 배율 계산

LightRoom 검색

방 내부 무작위 위치 선택

바닥 Raycast 검사

벽 여백 검사

플레이어 안전거리 검사

다른 오브젝트와의 충돌 검사

Trigger Collider 제외 처리



두 매니저가 동일한 밝기 기준과 위치 검색 규칙을 사용하도록 구성했다.



9\. Hierarchy 구조 정리



던전 Scene의 스폰 관련 오브젝트를 다음과 같이 분리했다.



SpawnManagers

├─ MonsterSpawnManager

└─ TreasureSpawnManager

몬스터는 MonsterSpawnManager의 자식으로 생성된다.

보물은 TreasureSpawnManager의 자식으로 생성된다.

기존 통합 SpawnManager 컴포넌트와 오브젝트를 제거했다.

각 매니저에 DungeonGenerator와 해당 프리팹 목록을 연결했다.

결과

던전 생성 완료 후 몬스터와 보물이 각각 자동 생성된다.

몬스터와 보물 생성 로직이 서로 독립된 컴포넌트로 분리됐다.

시작 방에는 몬스터와 보물이 생성되지 않는다.

어두운 방일수록 몬스터와 보물 생성량이 증가한다.

어두운 방에서 생성된 보물의 가치가 증가한다.

LightRoom Trigger 때문에 생성 위치가 모두 거부되던 문제가 해결됐다.

몬스터와 보물이 벽이나 다른 오브젝트와 겹치는 위치를 피한다.

생성된 몬스터와 보물을 Hierarchy에서 각각 구분해 확인할 수 있다.

이후 몬스터와 보물 스폰 규칙을 서로 독립적으로 확장할 수 있게 됐다.
