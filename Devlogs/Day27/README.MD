# 27일차 : 익스트랙션 결과 데이터

---

## 목표

- 던전 종료 원인을 하나의 결과 데이터로 확정한다.
- 마차 확보 정보와 던전 진행 정보를 결과에 기록한다.
- 다음 Scene과 28일차 마을 정산에서 결과를 사용할 수 있도록 유지한다.

---

## 한 일

- RunEndReason 종료 원인을 정의했다.
- RunResultData를 구현했다.
- RunResultManager를 구현했다.
- 결과 매니저를 DontDestroyOnLoad 방식으로 유지하도록 했다.
- Scene 전환 시 던전 관련 참조를 다시 연결하도록 했다.
- 마차 수동 탈출 결과를 기록하도록 했다.
- 제한시간 종료 시 자동 탈출과 유기 실패를 구분하도록 했다.
- 부활할 수 없는 플레이어 최종 사망 결과를 기록하도록 했다.
- 부활의 돌 사용 시 최종 사망 결과가 생성되지 않도록 했다.
- 마차의 전체 확보 아이템 수와 실제 보물 수를 구분하도록 했다.
- 확보한 보물 가치, 던전 진행시간과 생성 시드를 기록하도록 했다.
- 결과가 한 번만 확정되도록 중복 처리를 차단했다.
- 중앙 OnGUI 결과 화면을 추가했다.
- 기존 마차, 제한시간과 사망 화면이 중앙 결과 화면과 겹치지 않도록 수정했다.
- ManagerHub에 RunResultManager 참조와 검사를 추가했다.
- DungeonGenerator의 프리팹 누락 시 생성 상태가 고정되는 위치 문제를 수정했다.
- F3 키로 25일차 수직 슬라이스 검증 패널을 켜고 끌 수 있도록 수정했다.
- 검증 패널을 숨긴 상태에서도 F8 수동 재검증이 작동하도록 유지했다.
---

## 결과

- 수동 탈출, 자동 탈출, 유기 실패와 최종 사망이 서로 다른 결과로 기록된다.
- 마차에 확보한 보물 수와 가치가 결과 데이터에 유지된다.
- 던전 종료 후 결과가 다른 이벤트로 덮어써지지 않는다.
- 결과 데이터가 다음 Scene의 정산 시스템에서 사용할 수 있는 상태로 유지된다.
- 28일차 마을 정산과 빚 상환을 연결할 기반이 완성됐다.

---

## 현재 제한 사항

- 실제 마을 정산과 골드 지급은 아직 구현하지 않았다.
- 빚 잔액과 남은 기한은 아직 결과에 반영하지 않는다.
- 결과 화면은 임시 OnGUI 방식이다.
- 결과 Scene으로 자동 이동하지 않는다.
- 저장 파일에는 탐험 결과를 기록하지 않는다.
- 멀티플레이어 결과 합산과 플레이어별 생존 상태는 구현하지 않았다.
