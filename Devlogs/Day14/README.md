# 프로젝트 I — 14일차 개발 일지

단계: Phase 1 — 프로토타입(싱글 핵심 루프)

## 목표

플레이어 체력이 0이 되었을 때 사망 상태로 전환하고, 부활의 돌을 소지한 경우 아이템을 자동으로 소모하여 한 번 부활하도록 구현한다.

## 한 일

### 1. 플레이어 사망 상태 추가

- `PlayerController`에 `isDead` 상태 추가
- 체력이 0이 되면 사망 처리 실행
- 사망 후 추가 피해 무시
- 사망 시 수직 이동속도 초기화

### 2. 사망 중 조작 차단

- 플레이어 이동과 마우스 시점 입력 차단
- `PlayerCombat` 비활성화로 공격과 방패 방어 차단
- `PlayerInteractor` 비활성화로 상호작용과 아이템 버리기 차단
- 마우스 커서 잠금 해제

### 3. 부활의 돌 자동 소생

- 인벤토리에서 `부활의 돌` 이름의 아이템 검색
- 사망 시 부활의 돌 1개 자동 소모
- 최대 체력의 30%로 즉시 부활
- 부활의 돌이 없으면 사망 상태 유지

### 4. 방패 상태 초기화

- `PlayerCombat.OnDisable()` 추가
- 전투 컴포넌트 비활성화 시 방패 사용 상태 해제
- 남아 있는 피해 감소율을 0으로 초기화

### 5. 임시 사망 화면 추가

- 기존 `OnGUI` 방식으로 사망 안내 표시
- 부활의 돌이 없다는 안내 표시
- 플레이어 조작이 중단되었다는 안내 표시
- 디버그 UI 표시 여부와 관계없이 사망 화면 유지

### 6. 코드 주석과 형식 정리

- `PlayerController.cs`와 `PlayerCombat.cs`의 의미 있는 코드 줄에 한글 주석 추가
- 한 줄 조건문을 Allman 스타일로 정리
- 현재 구현 상태와 맞지 않는 오래된 설명 수정

## 결과

- 몬스터 공격과 낙하 피해로 플레이어 체력이 정상적으로 감소한다.
- 방패 사용 중에는 설정된 비율만큼 받는 피해가 감소한다.
- 부활의 돌을 소지한 상태에서 체력이 0이 되면 아이템이 소모되고 체력 30%로 부활한다.
- 부활의 돌이 없으면 사망 화면이 표시되고 플레이어 조작이 차단된다.
- 사망 상태에서 추가 공격을 받아도 체력이 변하지 않는다.

## 다음 예정

15일차에는 플레이어가 사망하거나 던전에 유기되었을 때 인벤토리에 있던 아이템을 월드에 드랍하는 기능을 구현한다.
