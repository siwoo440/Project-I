# 45일차 개발 일지

## 오늘의 개발 목표

마을에서 탐사 지역을 선택하고 실제 던전으로 출발할 수 있는 탐사 지역 선택 시스템을 구현한다.

탐험 종료 후 획득한 보물을 골드로 정산하고, 플레이어가 원하는 금액만큼 빚을 납부할 수 있는 마을 정산 시스템을 구성한다.

## 구현 내용

### 1. 탐사 지역 데이터 구조 구현

- `DungeonRouteData`를 이용한 탐사 지역 데이터 구조 구성
- 탐사 지역 10종의 개별 데이터 에셋 생성
- 지역별 고유 `Route ID` 설정
- 지역 이름과 설명 데이터 분리
- 위험 등급 데이터 연결
- 지역별 최소·최대 방 수 설정
- 지역별 마차 이동 비용 설정
- 지역별 함정 생성 배율 설정
- 지역별 던전 Scene 이름 설정
- 구현 여부에 따른 지역 활성화 상태 설정

### 2. 탐사 지역 데이터 추가

다음 10개 탐사 지역 데이터를 추가했다.

1. 왕가의 지하묘지
2. 잿빛 습지 수도원
3. 적맥 광산
4. 수문 도시
5. 검은가시 숲
6. 황혼의 천문원
7. 서리봉우리 요새
8. 잿불 대장간
9. 백골 궁전
10. 별이 잠든 심연

현재 실제 출발이 가능한 지역은 왕가의 지하묘지로 제한했다.

나머지 지역은 이후 전용 던전과 기믹이 구현된 후 순서대로 개방하도록 설정했다.

### 3. 위험 등급 데이터 구성

- 탐사 지역 위험 등급을 별도 데이터로 분리
- 지역 선택 화면에서 위험 등급 표시
- 위험 등급별 색상 구분
- 지역 데이터와 위험 등급 데이터 연결
- 등급 데이터의 재사용이 가능한 구조 구성

### 4. 탐사 지역 선택 관리자 구현

- 등록된 탐사 지역 목록 관리
- 기본 선택 지역 설정
- 지역 선택 상태 저장
- 선택한 지역 데이터 반환
- 이용할 수 없는 지역 선택 차단
- 지역별 마차 비용 검사
- 선택한 지역의 던전 Scene 이동 처리
- 중복 `Route ID` 검사
- 누락된 지역 데이터 검사
- 잘못된 데이터 발견 시 초기화와 출발 차단

### 5. 마을 탐사 지역 선택 UI 구현

- 마차 상호작용을 통한 지역 선택 화면 표시
- 탐사 지역 카드 목록 표시
- 여러 지역을 확인할 수 있는 페이지 이동 기능 추가
- 지역 이름과 설명 표시
- 위험 등급 표시
- 최소·최대 방 수 표시
- 마차 이동 비용 표시
- 구현되지 않은 지역의 출발 버튼 비활성화
- 선택한 지역 강조 표시
- 선택한 지역으로 출발하는 기능 추가
- 지역 선택 창 닫기 기능 추가

### 6. 왕가의 지하묘지 출발 연결

- 왕가의 지하묘지를 기본 선택 지역으로 설정
- 왕가의 지하묘지만 이용 가능 상태로 설정
- `Dungeon_Catacombs` Scene 연결
- Build Profiles의 Scene 목록 확인
- 마을에서 왕가의 지하묘지로 이동할 수 있도록 구성

### 7. 던전 방 수 생성 범위 보정

- 지역 데이터의 최소·최대 방 수를 던전 생성기에 전달
- 위층과 지하층 보장 방을 목표 방 수에 포함하도록 수정
- 보장 방 생성으로 최대 방 수를 초과하는 문제 방지
- 보장 방 생성 후 부족한 방만 추가하도록 보정
- 목표 방 수에 도달하지 못할 경우 경고 출력

### 8. 마을 정산 시스템 구현

- 탐험 종료 결과를 마을에서 확인하는 정산 창 구현
- 탐험 종료 원인 표시
- 회수한 보물의 골드 가치 표시
- 보물 판매 금액을 보유 골드에 반영
- 현재 보유 골드와 남은 빚 표시
- 플레이어가 직접 빚 납부 금액을 선택하는 기능 추가
- 소액·중간·대량 단위의 납부 금액 조절 버튼 추가
- 납부액을 0골드로 초기화하는 기능 추가
- 최대 납부 가능 금액 자동 선택 기능 추가
- 보유 골드보다 많은 금액 납부 방지
- 남은 빚보다 많은 금액 납부 방지
- 빚 납부 결과 표시
- 정산 완료 후 다음 날짜 진행
- 빚 전액 상환 시 캠페인 성공 상태 표시
- 상환 기한 초과 시 캠페인 실패 상태 표시

### 9. 정산 종료 후 플레이어 조작 복구

- 정산 창 표시 중 플레이어 이동 제한
- 정산 창 표시 중 상호작용 제한
- 정산 창 표시 중 전투 조작 제한
- 정산 창 표시 중 마우스 커서 표시
- 정산 확인 후 정산 창 종료
- 정산 창 종료 후 플레이어 조작 복구
- `VillagePlayerSpawner`와 정산 UI 참조 연결

### 10. Village Scene 구조 정리

- Village Scene의 중복 카메라 제거
- 플레이어 프리팹 카메라만 사용하도록 수정
- 활성 `MainCamera`가 하나만 존재하도록 정리
- 활성 `Audio Listener`가 하나만 존재하도록 정리
- 중복된 `VillageDungeonSelectionUI` 제거
- 마차에 연결된 지역 선택 UI 하나만 유지
- 마차와 플레이어 생성 위치 재확인
- 마을 시스템 오브젝트 연결 상태 정리

### 11. Audio Mixer 설정 수정

- 스크립트와 Audio Mixer의 노출 Parameter 이름 통일
- `MasterVolume` 확인
- `BGMVolume` 확인
- `SFXVolume` 수정
- `AmbientVolume` 수정
- 존재하지 않는 Parameter 경고가 발생하지 않도록 정리

## 최종 확인 항목

- 탐사 지역 10개의 `Route ID`가 서로 중복되지 않는지 확인
- 왕가의 지하묘지가 첫 번째 지역으로 표시되는지 확인
- 왕가의 지하묘지만 선택과 출발이 가능한지 확인
- 왕가의 지하묘지 선택 시 `Dungeon_Catacombs`로 이동하는지 확인
- 던전의 전체 방 수가 설정된 최소·최대 범위를 지키는지 확인
- 정산 창에서 보물 판매 금액이 골드에 반영되는지 확인
- 선택한 금액만큼 빚이 감소하는지 확인
- 정산 창 종료 후 플레이어를 다시 조작할 수 있는지 확인
- Village Scene의 활성 카메라가 하나인지 확인
- 활성 `Audio Listener`가 하나인지 확인
- 지역 선택 UI가 중복으로 표시되지 않는지 확인
- Audio Mixer Parameter 관련 경고가 발생하지 않는지 확인

## 45일차 결과

마을에서 탐사 지역 정보를 확인하고 왕가의 지하묘지를 선택하여 출발할 수 있는 기본 탐사 지역 선택 흐름을 구성했다.

탐험 종료 후 보물을 골드로 정산하고, 플레이어가 납부 금액을 직접 결정해 빚을 상환할 수 있는 캠페인 정산 흐름을 구현했다.

지역 데이터 중복, 잘못된 Scene 연결, 중복 카메라와 UI, Audio Mixer Parameter 불일치, 던전 최대 방 수 초과 가능성을 수정했다.

이후에는 각 탐사 지역 전용 Scene과 환경 기믹을 순서대로 제작하고, 지역 개방 조건을 캠페인 진행 상태와 연결해야 한다.
