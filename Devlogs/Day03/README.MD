# 프로젝트 I — 3일차 개발 일지

> **단계**: Phase 1 — 프로토타입 (싱글 핵심 루프)
> **목표**: 상호작용(E) 시스템 + 손 슬롯 1개(줍기/버리기) 구현 (기획서 PART 4.7, 5.4.3, 8.1)

---

## 오늘의 목표
카메라 전방 오브젝트를 감지해 **E로 상호작용**, 아이템을 **손 슬롯 1개**에 들고/내려놓는 기초 마련.
→ 밝기 시스템의 "빛(광원) vs 화력(무기)" 트레이드오프의 토대.

## 한 일

### 1. 상호작용 규격 — `IInteractable` 인터페이스
- `GetPrompt()` (안내 문구) + `Interact(PlayerInteractor)` (실행)
- 앞으로 레버·문·서랍·보물·아이템 등이 모두 이 인터페이스를 구현

### 2. `PlayerInteractor` (상호작용 감지기)
- 카메라 전방으로 **레이캐스트**(거리 3m)해 `IInteractable` 대상 감지
- **E** = 상호작용 / **Q** = 손 아이템 버리기
- 화면 중앙 **크로스헤어** + `[E] 줍기: …` 안내 표시(OnGUI 임시)

### 3. `PlayerHand` (활성 손 슬롯 1개)
- 한 번에 **하나의 아이템만** 손에 든다 (기획서 규칙)
- 카메라 앞 **HandAnchor**(자동 생성)에 아이템 부착
- `TryHold()` (손 비었을 때만) / `DropHeld()` (앞으로 던짐)

### 4. `PickupItem` (주울 수 있는 아이템)
- `IInteractable` 구현 → 상호작용 시 손에 들림
- 들면 물리/충돌 OFF + 앵커 부착, 버리면 물리 복구 + 앞으로 힘 적용
- Item Name 필드로 안내 문구 표시

## 결과
- 아이템을 바라보면 안내 표시 → **E로 줍기**, **Q로 버리기** 정상 동작 확인
- 손 슬롯 1개 규칙(들고 있으면 추가로 못 주움) 확인
