# 프로젝트 I — 4일차 개발 일지

> **단계**: Phase 1 — 프로토타입 (싱글 핵심 루프)
> **목표**: 인벤토리(5칸) + 무게 시스템 + 아이템 데이터(ScriptableObject) 도입 (기획서 PART 4.4, 8)

---

## 오늘의 목표
3일차의 "손 하나"를 **슬롯 보관 → 휠 선택 → 손에 표시** 구조로 확장하고, **무게가 이동·스태미너에 영향**을 주게 만든다. 아이템 수치를 **ScriptableObject**로 데이터화.

## 한 일

### 1. ItemData (ScriptableObject) 도입
- `Create → ProjectI → Item Data` 로 아이템 정의 에셋 생성
- 컬럼: 이름/설명/분류/무게/인벤칸/보너스칸(가방)/소음/배터리/상호작용/가격 — **아이템.csv와 대응** (5일차 CSV 임포트 발판)

### 2. InventorySystem 구현 (스텁 → 완성)
- **기본 5칸 + 가방 보너스 칸**(동시 적용), 아이템은 slots만큼 칸 차지
- **무게 합산** + `WeightRatio` + 구간별 페널티(기획서 PART 4.4.1):
  | 적재율 | 이동속도 | 스태미너 회복 |
  | :-- | :-: | :-: |
  | ~50% | 100% | 100% |
  | 50~80% | 85% | 75% |
  | 80~100% | 70% | 50% |
  | 100%+ (과적) | 40% | 10% |
- **마우스 휠로 선택 전환** → 선택 아이템을 손(PlayerHand.Anchor)에 표시, 나머지는 숨김
- 슬롯 초과 시 줍기 거부(`슬롯 부족` 로그)

### 3. 연동 리팩터링
- `PlayerHand`: 손 앵커 제공용으로 단순화 (선택 아이템 표시 위치)
- `PlayerInteractor`: 인벤토리 참조, **Q = 선택 아이템 버리기**
- `PickupItem`: ItemData 연동(무게·슬롯·이름), 저장/표시/배출 상태 관리
- `PlayerController`: **무게 페널티(이동속도·스태미너 회복) 반영**

### 4. 버그 수정
- `InventorySystem.cs`가 `Items/`·`Managers/` 두 곳에 중복 → **중복 정의 컴파일 에러** 발생 → `Items/`의 1일차 스텁 삭제
- 아이템 표시 이름이 fallback `"아이템"`으로만 뜨던 문제 → **DisplayName 로직 개선**(ItemData 이름 → 오브젝트 이름 순, 기본값 "아이템"은 미설정으로 간주)

## 결과
- 여러 아이템 줍기 → 슬롯 보관 → 휠 전환 → 손 표시, **무게 쌓이면 느려지고 스태미너 회복 저하**, 가방으로 칸 확장, 슬롯 초과 거부, Q 버리기 정상 확인
- 이름 표시 정상화
