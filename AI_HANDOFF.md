# AI 핸드오프 문서 — 프로젝트 I

> **이 문서의 목적**: 다른 AI(또는 새 대화 세션)가 이 프로젝트를 처음 접해도, 이 문서 하나만 읽으면 지금까지의 결정·현재 상태·코드 구조를 파악하고 바로 이어서 개발할 수 있게 하기 위함.
> **사용법**: 새 AI 세션을 시작할 때 이 파일 내용을 붙여넣거나, "이 레포의 `AI_HANDOFF.md`를 읽고 시작해줘"라고 요청하세요.
> **유지보수**: 매 개발일(Day) 종료 후, 이 문서의 "현재 상태" 섹션을 그날 작업 내용으로 갱신합니다. (개발 일지 `Devlogs/DayNN/`와 별개로, 이 문서는 항상 "최신 스냅샷"만 유지)

## 0. 사용자(개발자) 프로필 & 작업 방식 ★중요

- **1인 개발자**, 비개발자에 가까움 — **AI가 코드를 작성하고, 사용자는 Unity 에디터에서 GUI 조작(오브젝트 생성, 컴포넌트 부착, 값 입력, Play 테스트)을 수행**하는 방식으로 협업 중.
- 매 개발일(Day)마다 진행 방식:
  1. AI가 "오늘의 방향(간단 요약)" + "STEP 1, 2, 3…" 형태로 **하나하나 따라할 수 있게** 자세히 안내
  2. 사용자가 Unity에서 그대로 따라 하고 결과(정상/에러/이상 동작)를 보고
  3. AI가 문제를 고치거나 추가 요청을 반영
  4. 사용자가 "완료"라고 하면 → **개발 일지 작성 + git 커밋 + push**를 AI가 수행
- **커밋 메시지 규칙(필수, 고정 포맷)**: `n일차 : 작업 내용` — 전부 한글, 숫자는 아라비아 숫자. 예) `12일차 : 전투 시스템 구현`
- **개발 일지**: `Devlogs/DayNN/README.MD` 에 매일 작성 (목표/한 일/결과/다음 예정/참고 TODO 형식)
- AI는 스크립트를 직접 파일로 작성(Write/Edit)하고, 사용자는 Unity 에디터에서 오브젝트·컴포넌트·씬 세팅만 수행. **씬 파일(.unity) 내부는 AI가 직접 편집하지 않음** (전부 GUI 조작으로 안내).

## 1. 게임 기획 개요

- **게임명**: 프로젝트 I (가제)
- **장르**: 1인칭 협동 던전 탐험·생존·익스트랙션. 1~6인 멀티플레이. PC(Steam) 전용.
- **한 줄 컨셉**: 친구들과 어두운 던전을 탐험하고, 몬스터·함정을 피하거나 맞서며, 제한 시간 안에 보물을 들고 살아 돌아오는 협동 다크 판타지 게임.
- **레퍼런스**: Lethal Company, Dark and Darker, Deep Rock Galactic, Content Warning
- **세계관**: 저(低)마법 다크 판타지. 마법 체계 없음(위협=괴물+함정). 플레이어는 쇠락한 마을 주민들이 결성한 '도굴단'(호칭: 탐험가). 마을 부흥 + 생계를 위해 마차로 던전을 오가며 탐험.
- **★게임의 정체성 축 = 밝기 시스템**: 구역(방) 기본 밝기는 0(칠흑). 오직 광원(횃불/랜턴/횃불대)만이 빛을 만든다. 어두울수록 시야↓, 몬스터 스폰·공격성↑, 보물 스폰·가치↑ (하이리스크 하이리턴).
- **★핵심 목표(게임 오버 조건)**: 페이즈 기반 "빚 상환" — 각 페이즈마다 [목표 금액 + 기한], 기한 내 못 갚으면 게임 오버. 클리어할수록 금액·기한이 함께 증가.
- **핵심 루프**: 마을(준비) → 던전 선택(입장 전 랜덤 리스크 1~3개 확인) → 탐색(밝기 관리) → 보물 회수 → 마차 적재(확보 확정) → '떠나기'(익스트랙션) → 마을 정산 → 반복.

**기획서 전문**은 별도 위치에 있음(이 Unity 레포에는 없음): `D:\프로젝트 I\기획서\게임기획서.md` (PART 1~15, 전체 결정 사항 상세). 데이터 설계 원본: `D:\프로젝트 I\데이터\` (아이템/보물/몬스터 CSV + 정의서). **이 두 폴더는 Unity 프로젝트와 다른 드라이브(D:)에 있어 git 레포에는 포함되지 않음** — 필요 시 그 경로를 직접 참고.

이 Unity 프로젝트 안에는 기획서 요약이 없으므로, 데이터 스키마·수치는 `Assets/Data/*.csv`와 `Assets/ScriptableObjects/`의 실제 값을 우선 신뢰할 것.

## 3. 현재 개발 상태 (11일차까지 완료) ★가장 중요

> 최신 커밋: `f9686a0` (`11일차 : 몬스터 기본 AI …`). 상세 이력은 `git log --oneline`과 `Devlogs/Day01~11`.

### 완성된 시스템
1. **프로젝트 골격** (Day1): URP 프로젝트, 폴더 구조, 씬 4개, 매니저 스텁 11종, Git/LFS.
2. **1인칭 플레이어** (Day2): `PlayerController` — WASD 이동, 마우스룩, 달리기(Shift, 스태미너 소모), 점프(Space), 앉기(Ctrl), 체력100/스태미너100, 낙하 피해, 좌상단 디버그 표시.
3. **상호작용 & 손 슬롯** (Day3~4): `IInteractable`(공통 규격), `PlayerInteractor`(카메라 레이캐스트, E/Q), `ICarryable`(아이템·보물 공용), `InventorySystem`(5칸+가방보너스칸, 무게 페널티 구간별 이동속도·스태미너회복 저하, 마우스 휠 선택, 하단 중앙 핫바 OnGUI), `ItemData`(SO) + `ItemDataImporter`.
4. **밝기 시스템 ★핵심** (Day6): `LightSource`(휴대 광원, 초기 꺼짐, F로 토글 — **단, 토글은 '손에 든 것'만**, 소지만 하면 꺼지지 않고 계속 빛남), `Brazier`(고정 광원, 시간 경과 decay, E 재점화), `LightRoom`(구역 트리거, 기본밝기0+화톳불 합), `LightSystem`(최종밝기=방+소지한 모든 켜진 광원 합, 5단계, 상단 UI).
5. **던전 절차적 생성** (Day7): `Room`(4방향 벽 개폐), `DungeonGenerator`(그리드 랜덤워크, 방 프리팹 조립, 시드 지원). 방 프리팹에 LightRoom 내장 → 생성된 모든 방이 자동으로 밝기 시스템에 참여.
6. **보물 시스템** (Day8): `TreasureData`(SO) + `Treasure` — **가치는 생성 시점에 고정**(기본랜덤×밝기배수×리스크배수, 이후 불변 — 악용 방지 설계), **두손 운반**(대형 보물, 슬롯 대신 양손 점유, 들고 있으면 다른 것 못 주움), `TreasureDataImporter`.
7. **마차 & 익스트랙션** (Day9): `Wagon`(적재 목록·총 가치), `WagonCargo`(E=싣기), `WagonLever`(E=떠나기 → 던전 종료). **떠나기 시 마차 적재분 + 플레이어 인벤토리 전체를 함께 확보**(살아서 탈출하면 손에 든 것도 인정).
8. **던전 시간 시스템** (Day10): `DungeonTimeSystem` — 게임시간 6:00→24:00(현실 약 18분, 테스트 시 단축 가능), 우상단 시계 표시, 24:00 도달 시 기본은 유기(실패) 처리하되 **플레이어가 마차 근처(5m)에 있으면 자동으로 탈출 성공** 처리.
9. **몬스터 기본 AI** (Day11): `MonsterData`(SO, 체력/이속/공격력/사거리/쿨다운/시야/청각/어둠속공격성), `MonsterAI` — FSM(배회→추격→공격→복귀), 시각 탐지(시야각+거리+레이캐스트 가림 판정), 청각 탐지(플레이어 CharacterController 속도=소음, 달리면 먼 거리서도 들킴), **밝기 연동**(어두우면 CurrentBrightness<40일 때 탐지범위 1.5배). 몬스터의 플레이어 공격은 구현됨. **플레이어가 몬스터를 공격하는 기능은 아직 없음**(다음 작업).
