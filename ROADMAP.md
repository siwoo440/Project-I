# 프로젝트 I — 일차별 개발 계획 (Unity)

> 기획서 v1.2 기반. PART 13(기술)·PART 15(로드맵)·MVP 정의를 실제 개발 순서로 전개.
> **개발 형태**: 1인 전업 / **완성도 우선(마감 없음)**.
> ⚠️ **"일차"는 달력상 하루가 아니라 "하나의 작업 단위(세션)"** 이다. 난이도에 따라 실제로 며칠이 걸릴 수 있음. 앞 단계가 안정화되면 다음으로.
> ⚠️ **2026-07 개정**: 실제 개발(1~11일차)을 해보니 "하루=하나의 명확한 기능"으로 쪼갤 때 집중도가 훨씬 높았음. 이에 따라 12일차 이후를 블록 단위 → **전부 일차 단위로 재분해**함.

## 개발 대원칙
1. **수직 슬라이스 우선** — 폭보다 깊이. "1개 던전을 끝까지 도는 경험"을 먼저 완성.
2. **기능 먼저, 아트 나중** — 프리미티브/에셋 플레이스홀더로 로직부터.
3. **데이터 주도** — 수치는 코드가 아닌 ScriptableObject/CSV로.
4. **자주 커밋** — 각 일차 끝에 동작하는 상태로 Git 커밋 (`n일차 : 작업 내용`).
5. **하루 하나의 기능에 집중** — 여러 기능을 한 일차에 욱여넣지 않는다. 애매하면 쪼갠다.
6. **가장 큰 기술 리스크(네트워크·음성)는 싱글 루프 검증 직후 바로.**
